AI活用レポート

記載者：中馬 使用期間：2025/6/1～6/13

使用ツール

  • Cursor
  • Chat-GPT
  • bolt (https://bolt.new/)
  • Github Copilot
  • Stitch

使用用途

  • 開発時のコード補完
  • 相談
  • 単純作業依頼
  • PowerPoint作成
  • 個人開発

利用状況・具体例

ChatGPT

  • エージェントモードでの作業代行が非常に便利で、単純作業の依頼に強い。
  • ChatGPTとのチャット履歴をワークスペースで共有できるため、チーム内の思考トレースが容易になればという期待がある。

Bolt

  • Boltで開発を試みたが、「予期せぬエラー」が表示され、当日分の無料トークンが消し飛んだ。
  • 無料枠は商用利用不可、しかし自走力は高いと思う。
  • 溝部さん曰く、「勝手に色々やってくれるけど行きすぎる点がある」みたいな課題もある。

Github Copilot

  • 要件定義、設計段階からの開発に活用。
  • GitHub Codespaces（ブラウザ上の開発環境）と組み合わせて利用中。
  • これにより、macOS/Windowsといった環境差の影響が軽減される可能性が高いと感じている。（要インターネット）

Stitch　（https://stitch.withgoogle.com/projects/14620809550986074195?pli=1）

  • デザイン系AI。
  • Figmaと連携可能。
  • まだしっかりと利用できていないが、ささっとデザインを出してくれる。

Chrome 拡張（ChatGPT検索）

  • アドレスバー検索をChatGPTに設定できる拡張機能を試用。
  • 通常のウェブ検索よりもChatGPT検索のほうが使いやすいと感じる場面もあった。
  • 自分は検索に戻した。ChatGPT検索も便利だとは思う。

個人開発

  • 複数のシステムを公開中。知見を深めながら、ツール間の連携やプロンプト設計の改善に活かしている。
  • 特にコンテキストの設定、設計書などの配置や参照のさせ方を勉強中。

今後試したいこと

  • AWSの「Amazon Q Developer」：AWSサービスに特化したAIとして注目。
  • AWSに強いモデルということで、IAM設定やLambda開発との相性などを検証予定。

メリット

  • 個人開発は1日でシステム公開まで可能になった。
  • AIが提案してくれるサービスで初めて聞くものがあり、自分の引き出しを増やすことができた。

デメリット

  • 開発スピードが早すぎてコミットのタイミングを逃してバカでかコミットになってしまった。
  • コードレビューやテストをしっかり実施しないとAIに甘えた開発になってしまう、明確に決めた方が良いと感じた。

感想

  • このレポートもChatGPTにサポートしてもらっている。
  • AIに文章を書かせると箇条書きが多くなり、筆者の主観が無くなるので「AIが書いたな〜」と感じるようになった
  • 設計書など普段エクセルファイルに書いている内容もAIでどうにかできないか試してみたい。
